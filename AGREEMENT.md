# Agreement

This document outlines the agreed coding and Git conventions for the HamFocus project. All contributors are expected to follow these rules throughout development.

---

## CODE

1. **Give comments for each function or section of code** that you work on, to help others understand.
   > ex. `// Ini function untuk menambahkan button baru`

2. **Variable name → function of the variable**, use camelCase.
   > ex. `var actionButton`

3. **Commit and push for every change** that you do.

4. **For pull requests**, only do when you are finished with all your work in that feature.

5. **Follow separation of concern** — use the MVVM framework.

6. **Make the interface first together** as the *acuan* (reference) throughout making the app.

---

## GIT

1. **Use short, clear commit messages** to define your work to other developers.

2. **Follow commit conventions** from the GA (Fix, Refactor, etc.)
   > ex. `Fix: Add new task feature`

3. **Pull main to your branch** before making a pull request to main — make sure your branch is up to date.

4. **Branch name uses `Feature/...`** — ALWAYS PUSH TO BRANCH, NOT TO MAIN.

5. **Step-by-step workflow:**
   - Pull main
   - Do your job
   - Commit & push with a clear message
   - Pull main again (just to make sure)
   - Make pull request to main

6. **If there is a conflict**, make sure to resolve it in your own branch 

7. **If you want to rearrange folders**, ask Kak Yana first — **DO NOT CHANGE THE FOLDER STRUCTURE BY YOURSELF**.
